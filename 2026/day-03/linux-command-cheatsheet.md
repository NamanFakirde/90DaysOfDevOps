<h1>Linux Commands Practice</h1>
<p>
  <h3>Process Management</h3>
  This commands are used to monitor system performance, manage services and control running services.
  <ul>
    <li>top: Real-time display of system processes and resource usage.</li>
    <li>ps: Snapshot of current processes.</li>
    <li>kill: Used to terminate the process using its PID</li>
    <li>free: Display the disks free and used memory space.</li>
    <li>nohup: It stores the logs of any commands in a file.</li>
    <li>uptime: Shows how long the system has been running.</li>
    <li>watch: Runs a particular command repetedly at a given interval.</li>
  </ul>
</p>

<hr> 

<p>
  <h3>File System</h3>
  This commands are used to manipulate and interact with the file system.
  <ul>
    <li>ls: Lists the content of a current diectory.</li>
    <li>cd: Used to change the current working directory.</li>
    <li>rm: Used to remove/delete a file, <b>rmdir</b> is for deleting the directory.</li>
    <li>cat: Display the content of a file, <b>zcat</b> for zip file.</li>
    <li>head: returns starting 4-5 lines from the top of a file.</li>
    <li>tail: Returns bottom 4-5 lines from the file.</li>
    <li>cp: Used to copy the file from one place to another (source to destination).</li>
    <li>mv: This command is used to move the file from source to destination and also used for renaming the file.</li>
    <li>wc: Return lines,words,size of a file.</li>
    <li>echo: Used to write into the file.</li>
    <li>ln: create links between the files.</li>
    <li>chmod: Used to change & modify the file/directory permission.</li>
  </ul>
</p>

<hr> 

<p>
  <h3>Network trubleshooting</h3>
  This commands are used to monitor connectivity, check ports, and routing issues.
  <ul>
    <li>ping: check connectivity between the two nodes. </li>
    <li>iwconfig: Shows wireless connection to the system.</li>
    <li>netstat: Displays network connections, routing tables, and interface stats.</li>
    <li>curl : Transfers data from a serve.</li>
    <li>telnet: Checks if a specific port is open on a remote host.</li>
    <li>ip route: Used to view and configure the kernel routing table.</li>
    <li>mtr: Combines ping and traceroute into a real-time report</li>
  </ul>
</p>
